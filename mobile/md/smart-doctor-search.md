# Smart Doctor Search Roadmap

## 1. الهدف
1. جعل اختيار الطبيب عند المريض يعتمد على فهم ذكي لاحتياجه، وليس فقط مطابقة نصية بسيطة.
2. تقليل الوقت الذي يحتاجه المريض للوصول إلى الطبيب الأنسب.
3. جعل النتائج قابلة للتفسير عبر إظهار سبب الترشيح.

## 2. ما تم تفعيله حالياً
1. فهم النص الحر داخل صفحة الأطباء.
2. استخراج التخصص من البحث إذا كان موجوداً.
3. استخراج نمط الاستشارة:
   `أونلاين`
   `حضوري`
4. استخراج نية السرعة أو التوفر الحالي.
5. استخراج إشارات الترتيب مثل:
   `أفضل`
   `أرخص`
   `أقرب`
6. ربط النتائج بمحرك التوصية الموجود في السيرفر.
7. عرض نسبة المطابقة الذكية داخل بطاقة الطبيب.
8. عرض سبب مختصر للترشيح.

## 3. آلية العمل الحالية
1. المريض يكتب نص البحث.
2. التطبيق يمرر النص إلى محلل محلي داخل:
   `useSmartDoctorSearch`
3. المحلل يستخرج:
   specialization
   city / region
   consultationMode
   availableNow
   sortBy
4. هذه القيم تُرسل إلى endpoint الأطباء.
5. السيرفر يرتب النتائج بمحرك المطابقة الذكي.
6. الواجهة تعرض:
   ملخص كيف فُهم البحث
   الأطباء المرتبين
   سبب ظهور كل طبيب

## 4. المرحلة التالية المقترحة
1. دعم البحث بالأعراض:
   مثال `ألم صدر` أو `طفلي عنده حرارة`
2. عندها يتم إرسال النص إلى `recommended doctors` بدل البحث العام.
3. عرض `suggestedSpecialties` القادمة من السيرفر كمقترحات مباشرة للمريض.
4. إضافة زر:
   `ابحث حسب الأعراض`

## 5. مرحلة التحسين المتقدم
1. دعم التصفية بالسعر عبر slider.
2. دعم التصفية بالتقييم الأدنى.
3. دعم التصفية حسب:
   المتاح اليوم
   المتاح أونلاين فقط
   الأقرب جغرافياً
4. حفظ آخر عمليات البحث داخل الجهاز.
5. عرض:
   آخر بحث
   التخصصات الأكثر استخداماً
   الأطباء الذين فتحهم المريض سابقاً

## 6. مرحلة التخصيص الشخصي
1. ربط نتائج البحث ببيانات المريض:
   المدينة
   المنطقة
   التخصصات التي زارها سابقاً
   الحجوزات السابقة
2. ترجيح الأطباء القريبين أو المناسبين لحالة المريض بشكل شخصي.
3. تحسين ترتيب النتائج حسب سجل الاستخدام.

## 7. مرحلة الذكاء الأعلى
1. ربط البحث بنظام مرادفات عربية للتخصصات.
2. فهم الكلمات العامية أو المختصرة.
3. دعم البحث المختلط:
   `أفضل دكتور جلدية أونلاين ورخيص`
4. دعم اقتراح تلقائي أثناء الكتابة.
5. دعم تصحيح الأخطاء الإملائية في أسماء التخصصات والمدن.

## 8. ملاحظات هندسية
1. المنطق المحلي في الموبايل يجب أن يبقى خفيفاً ومفسراً فقط.
2. قرار الترتيب النهائي الأفضل يبقى في السيرفر.
3. أي توسعة مستقبلية يفضّل أن تبني فوق:
   `GET /doctors`
   `GET /doctors/filters`
   `GET /doctors/recommended`
4. عرض سبب الترشيح مهم بقدر أهمية الترتيب نفسه، لأنه يزيد ثقة المستخدم.

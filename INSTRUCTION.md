# تعليمات التعديلات المنفذة

## 1. تفعيل نظام مواعيد الطبيب
1. تم إنشاء جدول جديد في Prisma باسم `DoctorAvailabilitySlot`.
2. هذا الجدول يربط كل طبيب بالأيام والأوقات التي يحددها بنفسه.
3. تم إضافة العلاقة داخل `DoctorProfile` حتى يصبح لكل طبيب جدول حجز خاص به.
4. تم إنشاء migration جديدة باسم `0004_doctor_availability_slots`.

## 2. تحديث منطق السيرفر للحجوزات
1. تم إنشاء منطق لتجميع المواعيد المتاحة القادمة للطبيب من جدوله الأسبوعي.
2. تم استبعاد الأوقات التي تم حجزها مسبقاً من القائمة المعروضة للمريض.
3. تم التحقق في السيرفر أن الموعد المطلوب للحجز يطابق فعلاً جدول الطبيب.
4. تم منع الحجز إذا كان الوقت غير موجود ضمن أوقات الطبيب.
5. تم منع الحجز إذا كان الموعد محجوزاً مسبقاً.

## 3. إضافة endpoint للمواعيد المتاحة
1. تم إضافة endpoint جديد:

```txt
GET /api/v1/doctors/:id/appointment-slots?days=14
```

2. هذا المسار يرجع فقط المواعيد المتاحة للطبيب خلال عدد الأيام المطلوبة.
3. إذا لم يكن للطبيب جدول مواعيد، فالقائمة تكون فارغة.

## 4. تحديث تطبيق المريض
1. تم تعديل شاشة الحجز لتسحب المواعيد من السيرفر بدلاً من الأوقات الثابتة.
2. تم تقسيم المواعيد حسب اليوم وعرض الأوقات الخاصة بكل يوم.
3. تم جعل اختيار اليوم والوقت يملأ `appointmentDate` تلقائياً.
4. تم تعطيل زر تأكيد الحجز إذا لم يوجد موعد صالح.
5. تم إضافة حالة فارغة واضحة عندما لا توجد مواعيد متاحة للطبيب.
6. تم التعامل مع `404` في endpoint المواعيد على أنه "لا توجد مواعيد" بدلاً من خطأ عام.

## 5. تحسين تصميم شاشة الحجز
1. تم تصغير مربعات الأيام لتصبح أكثر تناسقاً.
2. تم تحويل اختيار الأيام إلى شريط أفقي قابل للتمرير.
3. تم الحفاظ على تمييز اليوم المختار بشكل واضح.

## 6. تحديث تطبيق الطبيب
1. تم تعديل شاشة `DoctorProfileScreen` لتصبح شاشة إدارة جدول الحجز.
2. أصبح الطبيب يستطيع اختيار الأيام والساعات المتاحة بنفسه.
3. تم حفظ جدول الطبيب في السيرفر مباشرة.
4. أي تعديل في جدول الطبيب ينعكس على شاشة الحجز عند المريض.

## 7. تحديث شاشة تفاصيل الطبيب
1. تم عرض معاينة للمواعيد المتاحة في صفحة تفاصيل الطبيب.
2. إذا لم توجد مواعيد، يتم تعطيل زر الحجز وإظهار رسالة مناسبة.

## 8. تحديث النماذج والـ hooks في الموبايل
1. تم إضافة أنواع جديدة للمواعيد:
   `DoctorAvailabilitySlotEntity`
   `DoctorAppointmentSlotEntity`
2. تم تحديث `DoctorRepository` و`DoctorService` لدعم:
   `getMyProfile`
   `updateMyProfile`
   `getAppointmentSlots`
3. تم إضافة use cases وhooks جديدة لجلب وتحديث مواعيد الطبيب.

## 9. تطبيق Prisma على قاعدة البيانات
1. تم تشغيل:

```bash
npm run prisma:deploy
```

2. ثم تم التحقق عبر:

```bash
npx prisma migrate status
```

3. النتيجة كانت أن قاعدة البيانات محدثة ولا توجد migrations معلقة.

## 10. التحقق النهائي
1. تم تشغيل `npm run typecheck` داخل تطبيق الموبايل ونجح بدون أخطاء.
2. تم تشغيل `npm run build` داخل السيرفر ونجح بدون أخطاء.

## 11. ملاحظة مهمة
1. نظام الحجز الآن يعتمد على الأوقات التي يحددها الطبيب بنفسه.
2. إذا لم يحدد الطبيب أي وقت، فلن تظهر للمريض أي مواعيد متاحة.
3. يمكن ضبط توقيت المنصة من خلال المتغير:

```txt
APPOINTMENT_TIMEZONE_OFFSET_MINUTES
```
